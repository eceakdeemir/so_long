# so_long

A small **2D tile-based game** written in C for the 42 curriculum. You move a character on a grid map, collect every item, then reach the exit. Rendering uses **MiniLibX** (Linux) with X11.

## Gameplay

- **Objective:** Collect all collectibles (`C`), then step on the exit (`E`) to win.
- **Mandatory build:** Move count is printed to the terminal after each step.
- **Bonus build:** Animated sprites for the player, collectibles, and enemies; move count is shown in the window; touching an enemy (`X`) ends the game in a loss.

## Controls

| Key | Action |
|-----|--------|
| `W` | Move up |
| `S` | Move down |
| `A` | Move left |
| `D` | Move right |
| `Esc` | Quit |

## Map format

Maps are plain text files with the `.ber` extension. Each line is a row; all rows must have the same length (rectangle).

| Character | Meaning |
|-----------|---------|
| `0` | Empty floor |
| `1` | Wall |
| `P` | Player start (exactly one) |
| `C` | Collectible (at least one) |
| `E` | Exit (exactly one) |
| `X` | Enemy — **bonus only** |

The map must be **fully enclosed** by walls on the outer border. The program validates the file (shape, character counts, valid path from the player to all collectibles and the exit via a flood-fill check). Invalid maps print an error and exit without opening a window.

Example maps in the repository: `map.ber`, `map2.ber`. `fail.ber` is useful for testing error handling.

## Requirements

- **OS:** Linux (the Makefile links against MiniLibX for Linux and system X11 libraries).
- **Tools:** `make`, `cc`, `curl`, `tar`.
- **Libraries:** X11 development libraries (`libX11`, `libXext`), `zlib`, `math` — as used in the link line in the `Makefile`.

> **Note:** This project is not set up for native Windows builds; use **WSL** or a Linux VM if you are on Windows.

## Building

The first build downloads and unpacks **minilibx-linux** into `mlx_linux/` (42 intra CDN URL in the `Makefile`).

```bash
make        # builds ./so_long
make bonus  # builds ./so_long_bonus
```

Other targets:

```bash
make clean   # removes binaries and runs clean in subprojects
make fclean  # clean + full clean of printf + removes mlx_linux
make re      # fclean then all
```

## Running

Pass exactly **one** argument: the path to a `.ber` map file.

```bash
./so_long map.ber
./so_long_bonus map2.ber
```

## Project layout

| Area | Role |
|------|------|
| `so_long.c`, `so_long.h` | Mandatory entry point and shared types |
| `actions.c`, `map_*.c`, `init_*.c`, `find_positions.c`, … | Game logic, parsing, validation, MLX setup, rendering |
| `so_long_bonus.c`, `so_long_bonus.h`, `*_bonus.c` | Bonus features (enemies, animations, on-screen counter) |
| `printf/` | Custom `ft_printf` used for logging |
| `get_next_line/` | Line-by-line map reading |
| `mlx_linux/` | MiniLibX (generated by `make`, removed by `fclean`) |
| `xpm/` | XPM sprite assets for walls, player, items, enemies, background |

The tile size used for drawing is **64×64** pixels (see `condition_image_func.c` / bonus equivalents).
